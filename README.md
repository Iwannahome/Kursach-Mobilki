# Kursach-Mobilki
# Курсовая Работа Архитектура операционных систем мобильных устройств 
# На тему Сборка прошивки мобильного устройства на базе AOSP версии android-9.0.0_r38
Актуальность темы
Android сегодня занимает более 70% мирового рынка мобильных ОС.
Производители устройств постоянно адаптируют систему под собственное
аппаратное обеспечение, что приводит к фрагментации и задержкам обновлений.
Проект Treble и технология Generic System Image (GSI) призваны решить эти
проблемы, обеспечивая единую модульную архитектуру и унифицированные
образы ОС для множества устройств. Понимание внутреннего устройства
Android и процесса сборки GSI-образов критично для разработчиков и
системных инженеров.
Производители устройств вынуждены разрабатывать и адаптировать код ОС для
каждой своей аппаратной платформы независимо, что приводит к значительному
замедлению распространения обновлений. Эта проблема усугубляется
необходимостью согласования всех изменений с Google и соблюдением
множества требований совместимости. В результате среднее время между
выпуском новой версии Android и её доступностью на конкретном устройстве
составляет от нескольких месяцев до нескольких лет.
В ответ на эту проблему Google в 2017 году инициировала проект Project Treble,
кардинально изменивший архитектуру Android. Проект вводит принцип
модульности, разделяя программное обеспечение устройства на два четко
определенные слоя: универсальный системный слой (System Partition) и
специфичный для оборудования вендорский слой (Vendor Partition). Этот
архитектурный сдвиг создал основу для технологии Generic System Image (GSI)
— универсальных системных образов, которые могут работать на любом
устройстве, поддерживающем Project Treble, независимо от конкретного
аппаратного обеспечения.
Понимание процесса сборки, модификации и установки GSI-образов является
критически важным навыком для разработчиков, системных инженеров,
инженеров по качеству и энтузиастов, желающих углубить свои знания в области
встраиваемых систем и мобильной архитектуры. Данная курсовая работа
предоставляет возможность получить практические навыки в этой области,
применяя полученные знания к реальной платформе.
Цель работы
Получение практических навыков сборки и адаптации Android 9.0.0-38 из
исходного кода AOSP для устройства Khadas Vim 3L.
Задачи
1. 2. Проанализировать структуру Android Open Source Project (AOSP).
Исследовать архитектуру Project Treble и принципы GSI.
3. Изучить особенности Android 9.0.0-38 (Pie) и её технические
параметры.
4. Освоить процесс подготовки, сборки и установки GSI-образа.
5. Выполнить полный цикл диагностики целевого устройства Khadas
VIM 3L, включая анализ текущего программного обеспечения, версий
компонентов и структуры разделов памяти
6. Успешно установить модифицированный образ на целевое
устройство и провести полную валидацию функциональности системы
Объект и предмет исследования
Объект: операционная система Android 9.0.0-38.
Предмет: процесс адаптации и сборки GSI-образа под устройство Khadas Vim 3L.
Методы
Анализ документации и исходных кодов AOSP; экспериментальная сборка;
тестирование на реальном устройстве.
